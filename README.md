# PDF Direct Writer Web v5

## v5 修正点

### 1. 折返し右端待機中の Enter 動作

折返し右端クリック待ち中に Enter を押した場合は、

```text
現在の開始位置からページ右端まで
```

を折返し幅として確定します。

Esc を押すと入力をキャンセルします。

### 2. 「右をクリックしているのに注意が続く」問題への対策

v4 fixed では、折返し幅が 20pt 未満の場合に警告していました。
この判定が厳しすぎるため、v5では 5pt に緩和しました。

### 3. ガイドの視認性向上

折返し右端待機中にマウスを動かすと、ガイド線が現在のマウス位置まで伸びます。

## 起動方法

```powershell
pip install -r requirements.txt
python app.py
```



## v6 修正点

### 1. 文字サイズ不正入力時の戻り方を改善

v5では、文字サイズに半角数字以外を入力すると、アラート後に追加処理がキャンセルされました。

v6では、以下の流れに変更しました。

```text
文字入力済み
↓
文字サイズで不正入力
↓
アラート表示
↓
文字サイズ入力だけ再表示
```

開始位置クリックや文字入力までは戻りません。

### 2. output フォルダへの出力について

`PDF出力` ボタンを押すと、サーバー側で以下の場所にPDFが保存されます。

```text
output/direct_result_<session_id>.pdf
```

その後、ブラウザが `/download/<filename>` に移動し、ダウンロードが開始されます。

つまり、現在の仕様では、

```text
outputフォルダへ保存
↓
同じPDFをブラウザからダウンロード
```

という流れです。

ローカル開発中は output フォルダにもPDFが残ります。


## v7 修正点

### 折返し右端待機UIを大型化

v6では、文字入力・文字サイズ入力と比べて、
折返し右端待機UIが小さく、UIリズムが分断されていました。

v7では：

- フローティングUIを大型化
- 文字サイズを拡大
- Enter / Esc の操作も表示
- 半モーダル風デザインへ変更

これにより、

文字入力
↓
文字サイズ入力
↓
折返し右端設定

の流れが自然につながるよう改善しました。
